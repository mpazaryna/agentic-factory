# 2026-03-13: Skills 2.0 Migration

Major restructure of the entire agentic-factory repo, driven by reading Rick Hightower's article on Claude Code Agent Skills 2.0.

## What Happened

Started by reading the Skills 2.0 PDF — the key insight was that commands and skills are now unified. Every `SKILL.md` with a `name` field is a slash command. No more `commands/` wrapper layer.

### Migration sequence (domain by domain):

1. **senior-pm** — Split monolith `skills/senior-pm/SKILL.md` into 3 focused skills (portfolio-health, risk-analysis, resource-planning). Removed redundant `commands/senior-pm.md` wrapper. Tested install from local marketplace — worked.

2. **clickup** — Converted 4 commands to skills. Added `context: fork` to the autonomous agent skill. Added `disable-model-invocation: true` to all workflow skills. Made conventions `user-invocable: false`. Tested install — worked.

3. **platform-tools** — Broke the grab-bag into 4 independent plugins (cloudflare, fork-terminal, goose, uat-audit). Cloudflare monolith became 5 background skills. Updated marketplace with all new entries.

4. **developer-workflow + codebase-analysis** — Moved under `dev/`. Converted all commands. Added `context: fork` for research-task and technical-decision. Git skills got `disable-model-invocation: true`.

5. **prompts/** — Promoted PKM to `pm/pkm/` plugin (8 skills). Promoted yoga to `domain/yoga/` plugin (5 agents). Folded anti-slop into content as background skill. iOS prompts became a swift-lang reference.

6. **private/** — Deleted entirely. Project-specific tools belong in project repos.

7. **Renamed** — content-creation → content, project-management → pm, platform-tools → platform.

8. **Added orchestra plugin** — Extracted the `.orchestra/` methodology from chiro-base into a reusable plugin with scaffold, roadmap, and conventions skills.

### Final tally:
- 15 plugins across 6 groupings (dev, product, content, platform, pm, domain)
- Zero `commands/` folders
- Zero `prompts/` folder
- Zero `private/` folder
- Marketplace works from GitHub: `mpazaryna/agentic-factory`

## Key Decisions

- `context: fork` for autonomous and research skills — keeps main conversation clean
- `disable-model-invocation: true` for anything with side effects (git, ClickUp API, scaffolding)
- `user-invocable: false` for background knowledge (cloudflare services, anti-slop, conventions)
- Top-level folders are organizational only — plugins always one level down
- Skill names describe function, never repeat plugin name

## What I Learned

The Skills 2.0 frontmatter fields (`context`, `disable-model-invocation`, `user-invocable`, `allowed-tools`, `agent`) aren't just configuration — they're the control plane for how skills behave. Getting these right is what separates a skill that surprises you from one that does exactly what you intended.

The biggest win is `context: fork`. The clickup agent skill used to eat the entire conversation context when running autonomously. Now it runs in its own 200k-token window and returns a summary. That alone was worth the migration.
