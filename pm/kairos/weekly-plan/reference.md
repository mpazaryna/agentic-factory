# Weekly Plan Reference

## Load Thresholds

| Pomodoro Avg | Level | Notes |
|--------------|-------|-------|
| <= 5 | Recovery | Light, review-driven |
| 6-9 | Steady | Standard operating mode |
| 10+ | Heavy | Max 2 consecutive weeks |

**Constraints:**
- Heavy is time-limited: max 2 consecutive weeks
- After 2 heavy weeks, next week must be steady or recovery
- Illness flag forces recovery regardless of other metrics

## Project Status Questions (used only when flagged)

| Status | Question |
|--------|----------|
| active | What's the commitment? (cap it) |
| encouraged | What specific task? (name it or defer intentionally) |
| stable | No action unless it breaks |
| paused | Unblock or archive? |
| concept | Start or archive? |

## Weekly Summary Evaluation (for /weekly-summary)

**Score coverage:**
- Did every milestone in the score get attention, or did Professional crowd everything out?
- Were non-code commitments (lessons, classes, yoga, travel planning) honored?

**Frog compliance:**
- How many frogs were named at shutdown?
- How many were eaten the next morning?
- Any frog carry over 2+ days?

**ClickUp alignment:**
- Were MUST DO tasks completed as assigned?
- How many tasks carried over vs completed on their assigned day?
- Any tasks that sat unworked all week?

## ICS Time Block Rules

- Work blocks only. No lunch, transitions, routines, wind-down, shutdown.
- Actual task names. "Chiro: R-code negation bug (URGENT)" not "Block 1: Chiro"
- Non-code items get blocks too. "Piano: lesson" or "Art: class" are real blocks.
- ClickUp task ID in description. Everything has a ClickUp task.
- Monday frog is named. Tue-Fri frogs named at prior day's shutdown.
- Standard day shape: Morning blocks start 8:30, afternoon blocks start 1:30, end by 4:00.
- The ICS is a notification layer, not the source of truth.

## Clockify Naming Convention

Clockify project names MUST match ClickUp list/folder names exactly. No catch-all projects.
