# Validate Command

> Score Claude Code components against quality rubrics

## Quick Start

```
/validate skill skills/yoga-class-planner
/validate agent agents/quality-control-enforcer
/validate command commands/gh-issue
```

## Overview

The `/validate` command evaluates skills, agents, commands, and curated prompts against structured quality rubrics. It produces a detailed scorecard with PASS/FAIL per check and a total score out of 100.

## What This Command Does

- Reads all files in a component directory
- Evaluates against a type-specific rubric (structure, frontmatter, content quality, documentation, cleanliness)
- Scores each check as PASS or FAIL with point values
- Produces a formatted scorecard with a rating (EXCELLENT / GOOD / NEEDS WORK / POOR)
- Provides actionable recommendations

## Supported Component Types

| Type | Example Path |
|------|-------------|
| `skill` | `skills/yoga-class-planner` |
| `agent` | `agents/research-agent` |
| `command` | `commands/gh-issue` |
| `curated-prompt` | `curated-prompts/engineering/market-research` |

## Output Format

- Per-category breakdown with earned/possible points
- Individual PASS/FAIL for every check
- Total score out of 100 with rating
- Actionable improvement recommendations

## When to Use

- After building a new component with `/build`
- Before sharing or publishing a component
- To audit existing components for quality
- As part of a build-validate-install workflow
